## Detection Targets

These apps are designed to test the `harden` CLI's ability to detect:

### External Services
- **Databases**: MongoDB, PostgreSQL, SQLite, Elasticsearch
- **Message Queues**: Redis, Kafka, Celery
- **Cloud Storage**: GCS, Azure Blob Storage, AWS S3
- **AI/ML Services**: HuggingFace, Together AI
- **Enterprise APIs**: Microsoft Graph, Azure AD

### Security Issues
- **Credentials**: Connection strings, API keys, client secrets
- **Vulnerabilities**: SQL injection, SSRF, path traversal
- **Configuration**: Debug mode, missing authentication, no rate limiting
- **File Issues**: .env files committed, secrets in plaintext

### Infrastructure Patterns
- Background task processing (Celery)
- Event streaming (Kafka)
- Caching layers (Redis)
- Full-text search (Elasticsearch)
- File storage services
- OAuth2 authentication flows
